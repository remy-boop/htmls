Detailed step-by-step flow
1
CX agent runs /fetch-reviews in Slack
Command typed in #app-reviews-ai channel with parameters to control what reviews to fetch.
/fetch-reviews ratings:1,2 days:7 country:MX max:5
2
Slack sends the command to Render via HTTP POST
Slack calls the bot's /slack/commands endpoint on Render. The server wakes up (may take 30–60s on free tier cold start), then parses all the params.
Slack Bolt SDK Node.js / Express on Render
3
Server fetches reviews from Google Play
Authenticated via Service Account JSON from Google Cloud. Fetches reviews matching the ratings and days filters directly from the API.
Google Play API · googleapis npm Reply speed: instant
4
Server fetches reviews from App Store
Authenticated via short-lived JWT (RS256, 20 min expiry). Reviews are fetched from App Store Connect API, then filtered by territory client-side using the attributes.territory field — Apple does not support filter[territory] as a query param.
App Store Connect API · JWT RS256 Reply speed: 2–24 hrs (Apple review queue)
5
Neon DB filters out already-answered reviews
Server checks the sent_replies table in Postgres. Any review that already has a sent reply is dropped. Only genuinely unanswered reviews move forward.
Neon Postgres · sent_replies table
6
Server saves new reviews to DB
Each new review is written to the reviews table. A draft slot is created in response_drafts, ready to receive the AI response.
Neon Postgres · reviews + response_drafts tables
7
Claude API generates one draft reply per review
Server calls the Anthropic API with the review text plus knowledge-base.md loaded as context. Claude writes a brand-appropriate response, matched to the review's language.
claude-sonnet-4-5 · ~$0.01/review knowledge-base.md
8
Bot posts each review to #app-reviews-ai in Slack
One Slack message per review. Shows: store, rating, review text, AI draft response, and two action buttons. Slack message ID is stored in slack_messages table for routing button clicks.
✓ Approve button ✕ Reject button slack_messages table
9
CX agent reads the draft and decides
Agent reads the review and AI draft in Slack. They click Approve to publish the reply as-is, or Reject to skip it. No inline editing yet — reject means discard.
10
If Approved → server posts reply to the store
Slack sends the button click to /slack/actions endpoint. Server looks up the review, calls the store API to submit the reply, then writes the result to sent_replies so it won't appear again.
Google Play · instant App Store · 2–24 hrs (Apple review queue) sent_replies table updated
✕
If Rejected → review is marked skipped in DB
No reply is sent to the store. The review is recorded as skipped in sent_replies so it won't surface again on the next /fetch-reviews run.

knowledge-base.md — AI context file
Lives on Render at project root. Loaded at server startup. Controls Claude's tone, language detection, escalation rules, and TREBEL brand voice. Edit the file and redeploy to update AI behavior without touching code.
Neon DB schema
TableWhat it stores
reviews Raw review — store, rating, country, language, date, review ID
response_drafts AI-generated reply text per review, linked to review ID
sent_replies What was sent, when, and by which Slack user (used for dedup)
slack_messages Slack message ID per review for button interaction routing
