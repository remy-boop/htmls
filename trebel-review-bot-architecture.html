Detailed step-by-step flow
1
CX agent runs /fetch-reviews in Slack
Command typed in #app-reviews-ai channel with parameters to control what reviews to fetch.
/fetch-reviews ratings:1,2 days:7 store:both country:mx max:10
ratings — star filter days — lookback (keep ≤30) store — google_play / app_store / both country — ISO code, comma-separated max — 1–100, default 20
2
Slack sends the command to Render via HTTP POST
Slack calls the bot's /slack/commands endpoint on Render. The server wakes up (may take 30–60s on free tier cold start), then parses all the params.
Slack Bolt SDK Node.js 18 + TypeScript on Render
3
Server fetches reviews from Google Play
Authenticated via Service Account JSON from Google Cloud. Fetches reviews matching the ratings and days filters directly from the API.
Google Play API · googleapis npm Reply speed: instant
4
Server fetches reviews from App Store
Authenticated via short-lived JWT (RS256, 20 min expiry). Reviews fetched without territory filter — Apple does not support filter[territory] as a query param. Territory is filtered client-side via attributes.territory after fetching all pages. Full pagination via links.next cursor. 180+ ISO 3166-1 codes mapped to Apple's 3-letter format (e.g. mx → MEX, us → USA).
App Store Connect API · JWT RS256 Client-side territory filter Reply speed: 2–24 hrs (Apple review queue)
5
Neon DB filters out already-answered reviews
Server checks the sent_replies table in Postgres. Any review that already has a sent reply is dropped. Only genuinely unanswered reviews move forward.
Neon Postgres · sent_replies table
6
Server saves new reviews to DB
Each new review is written to the reviews table. A draft slot is created in response_drafts, ready to receive the AI response.
Neon Postgres · reviews + response_drafts tables
7
Claude API generates one draft reply per review
Server calls the Anthropic API with the review text plus knowledge-base.md loaded as context. Claude writes a brand-appropriate response, matched to the review's language. Reply capped at 350 characters (Google Play hard limit).
claude-sonnet-4-5 · ~$0.01/review knowledge-base.md 350 char limit enforced
8
Bot posts each review card to #app-reviews-ai in Slack
One Slack message per review. Shows: store, rating, review text, AI draft response, and three action buttons. Slack message ID stored in slack_messages table for routing button clicks and updating the card after action.
✅ Approve & Send ✏️ Edit ❌ Reject slack_messages table
