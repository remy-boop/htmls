Step-by-step walkthrough

End-to-end flow from slash command to reply posted in the store.

1

CX agent runs /fetch-reviews in Slack

Command typed in #app-reviews-ai with parameters controlling what reviews to fetch.

/fetch-reviews store:both country:mx ratings:1,2 days:7 max:2
ratings — star filter days — lookback (keep ≤30) store — google_play / app_store / both country — ISO code, comma-separated max — 1–100, default 20
2

Slack sends the command to Render via HTTP POST

Slack calls the bot's /slack/commands endpoint on Render. The server wakes up (may take 30–60s on free tier cold start), then parses all params.

Slack Bolt SDK Node.js 18 + TypeScript on Render
3

Server fetches reviews from Google Play

Authenticated via Service Account JSON from Google Cloud. Fetches reviews matching the ratings and days filters directly from the API.

Google Play API · googleapis npm GET 200 req/hr POST 2,000 req/day Reply speed: instant
4

Server fetches reviews from Apple App Store

Authenticated via short-lived JWT (RS256, 20 min expiry). Reviews fetched without territory filter — Apple does not support filter[territory] as a query param. Territory filtered client-side via attributes.territory after fetching all pages.

App Store Connect API · JWT RS256 GET ~3,600 req/hr POST shared GET pool Client-side territory filter Reply speed: 2–24 hrs
5

DB filters out already-answered reviews

Server checks the sent_replies table in Postgres. Any review that already has a sent reply is dropped. Only genuinely unanswered reviews move forward.

Neon Postgres · sent_replies table
6

Server saves new reviews to DB

Each new review is written to the reviews table with replied=false. A draft slot is created in response_drafts, ready to receive the AI response.

Neon Postgres · reviews + response_drafts
7

Claude generates one draft reply per review

Server calls the Anthropic API with the review text plus knowledge-base.md loaded as context. Claude writes a brand-appropriate response, matched to the review's language. Reply capped at 350 characters (Google Play hard limit).

claude-sonnet-4-5 · ~$0.01/review knowledge-base.md 350 char limit enforced
8

Bot posts each review card to #app-reviews-ai

One Slack message per review. Shows store, rating, review text, AI draft response, and action buttons. Slack message ID stored in slack_messages table for routing button clicks and updating the card after action.

✅ Approve & Send ✏️ Edit
slack_messages table · msg ID stored
9

CX agent reads the draft and decides

Approve sends the AI draft as-is. Edit opens a Slack modal with the draft pre-filled — agent edits and submits. Server enforces 350-char limit on submission.

10

Render posts reply to store

Slack sends the button click or modal submission to /slack/actions endpoint. Render looks up the review, calls the store API to submit the reply, writes to sent_replies, and updates the Slack card to ✅ Sent (by @user).

Google Play: instant Apple App Store: 2–24 hrs sent_replies table updated Slack card updated via msg ID
